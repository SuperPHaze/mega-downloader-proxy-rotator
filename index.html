Mega Downloader Proxy Rotator

A desktop app that splits the download into many small chunks and routes each one through a different proxy, in parallel. So the daily per-IP limit that blocks multi-GB files no longer stops you halfway through.

A couple of words from the person who wrote it

This is my first public project on GitHub. It started from a very concrete need of mine: accessing the files I keep on Mega from any computer — the one at home and the one at work — without logging in every time.

The thing is, without logging in Mega lets you download 5 GB a day: once that's gone, you have to wait 24 hours. When I need to move more of my own stuff, that gets inconvenient. That's where this came from.

At some point it stopped being an experiment and became something I used every day, so I cleaned it up and put it online. It's a first for me too: if you find a bug or have an idea, open an issue — I'll gladly read it.

The solution

A different proxy for every chunk

Instead of pulling everything from a single address, the file is split and then reassembled: each chunk goes through a different public proxy. The load spreads across many IPs and none of them gets put under pressure.

⚡

Built for multi-GB files

Handles downloads that would stall from a single IP, putting the file back together piece by piece down to the last one.

🔄

It finds the proxies for you

The app collects and verifies thousands of public proxies on its own and rotates through them while downloading. You configure nothing.

⏸

Resumes where it left off

Crash, dropped connection, computer turned off: on restart it only recovers the missing chunks, it doesn't start over from scratch.

🖥️

An interface that makes sense

Paste the link, hit start. See speed, estimated time, and the status of each file, with a light or dark theme.

🧩

Several files at once

Download multiple files together, each with its own queue of chunks and its own proxies, in parallel.

🔒

Doesn't show your IP

The service sees the proxies' IPs, not yours. Free proxies, though, don't guarantee anonymity — more on that below.

Under the hood

How it's built inside

The technical choices that keep downloads fast and reliable despite the inherent instability of free proxies. The specs, in short.

🧱 Fixed-size chunk queue

The file is split into 8 MB chunks, downloaded with parallel HTTP Range requests — 4 connections by default, up to 5 files at once. The fixed size makes resume valid regardless of the number of connections; a dropped proxy costs at most one chunk.

🔄 Per-chunk rotation + watchdog

Rotation is per-chunk, not per-file: each chunk uses a different proxy from the pool. A watchdog aborts attempts below 200 KB/s or past 180 s and retries them elsewhere, so slow or blocked proxies don't drag down the whole download.

✅ Two-stage validation

Stage 1 — HTTP reachability, up to 200 parallel checks with aggressive timeouts. Stage 2 — reachability of Mega's API host. Out of up to 1,000 candidates, usually only a few dozen get through: about 70% are discarded.

🏅 Scored pool

Every proxy has a score: +5 on success, −10 on failure, discarded below threshold. Hard penalties for CDN rejections (403/509/503), soft ones for transient errors; ties are broken by lower latency. A background refresher regenerates it without stopping downloads.

💾 Resume and atomic writes

It always writes to a .part file with a .progress.json sidecar tracking completed chunks; the rename to the final name is atomic and is the only completion marker. On resume, only the missing chunks are re-downloaded.

🔐 Built-in AES decryption

Mega's AES primitives are reimplemented in the project, with no fragile external dependencies. Decryption happens in streaming fashion (AES-CTR) straight onto the file: constant RAM even for files of many GB.

In three steps

How to use it

01

Paste the link

Copy one or more mega.nz/… links and paste them into the app, or import them from a text file.

02

Press start

The app looks for proxies, verifies them, and fills its reservoir. You don't have to find or enter any proxy by hand.

03

Download

The download starts in parallel chunks. Follow progress and speed; you can pause, cancel, or resume.
